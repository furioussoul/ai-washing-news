# 第11课：用AI开发一个小程序

> 课程模块：实战开发 | 预计时长：60分钟

---

## 一、课程导入

### 为什么学小程序

网页应用做好了，但用户需要打开浏览器、输入网址才能访问。

小程序的优势：
- 即用即走，无需下载
- 微信生态，流量巨大
- 开发成本低，一套代码多端运行

**这节课，我们用AI开发一个实用的小程序。**

---

## 二、核心内容

### 2.1 小程序技术选型

| 方案 | 优点 | 缺点 |
|-----|------|------|
| 微信小程序原生 | 性能最好 | 只支持微信 |
| uni-app | 一套代码，多端运行 | 性能略差 |
| Taro | React语法，开发体验好 | 学习成本 |

**本课选择**：uni-app（Vue语法，简单易学，支持多平台）

---

### 2.2 项目：记账小程序

#### 需求概述

一个简单的个人记账工具：
- 记录收入和支出
- 按分类统计
- 月度报表
- 数据本地存储

#### 页面设计

```
页面列表：
1. 首页（tabBar）
   - 本月收支概览
   - 最近记录列表
   - 快速记账入口

2. 记账页（tabBar）
   - 金额输入
   - 收支类型选择
   - 分类选择
   - 备注输入
   - 保存按钮

3. 统计页（tabBar）
   - 月度收支趋势图
   - 分类占比饼图
   - 支出排行榜

4. 我的（tabBar）
   - 数据导出
   - 分类管理
   - 关于
```

---

### 2.3 开发流程

#### 步骤1：项目初始化

**给AI的需求**：
```
使用uni-app创建记账小程序项目：
1. 使用Vue 3 + TypeScript
2. 配置uni-ui组件库
3. 项目结构：
   - pages/ 页面文件
   - components/ 公共组件
   - utils/ 工具函数
   - static/ 静态资源
4. 配置tabBar：首页、记账、统计、我的

输出：完整的项目结构和配置文件
```

---

#### 步骤2：首页开发

**给AI的需求**：
```
开发记账小程序首页：
1. 顶部显示本月收支概览
   - 本月收入（大字体）
   - 本月支出（大字体）
   - 结余

2. 最近记录列表（显示最近10条）
   - 分类图标
   - 分类名称
   - 金额（收入绿色，支出红色）
   - 时间

3. 底部快速记账按钮（悬浮）

使用uni-ui组件，风格简洁。
```

---

#### 步骤3：记账页开发

**给AI的需求**：
```
开发记账页面：
1. 金额输入区
   - 大字体显示金额
   - 数字键盘

2. 类型切换
   - 支出 / 收入 切换

3. 分类选择
   - 网格展示分类图标
   - 支出分类：餐饮、交通、购物、娱乐等
   - 收入分类：工资、兼职、投资等

4. 备注输入
   - 文本输入框
   - 快捷标签（早餐、午餐、晚餐等）

5. 日期选择
   - 默认今天
   - 可选择其他日期

6. 保存按钮
   - 验证金额>0
   - 保存到本地存储
   - 返回首页
```

---

#### 步骤4：数据存储

**给AI的需求**：
```
实现本地数据存储：
1. 使用uni.setStorageSync存储记账记录
2. 数据结构：
   {
     id: 唯一标识,
     amount: 金额,
     type: 'expense' | 'income',
     category: 分类ID,
     note: 备注,
     date: 日期(YYYY-MM-DD),
     createdAt: 创建时间
   }

3. 实现CRUD操作：
   - addRecord: 添加记录
   - getRecords: 获取记录列表（支持按月份筛选）
   - updateRecord: 更新记录
   - deleteRecord: 删除记录

4. 数据备份到云端（可选）
```

---

#### 步骤5：统计页开发

**给AI的需求**：
```
开发统计页面：
1. 月份选择器
   - 选择查看哪个月的数据

2. 收支趋势图
   - 使用ucharts或类似库
   - 显示每日收入/支出趋势

3. 分类占比饼图
   - 显示各分类支出占比
   - 点击可查看详情

4. 支出排行榜
   - 按金额排序的分类列表
   - 显示占比进度条

5. 月度汇总
   - 总收入、总支出、结余
```

---

### 2.4 发布流程

#### 微信小程序发布

1. 注册微信小程序账号
2. 获取AppID
3. 在HBuilderX中配置AppID
4. 点击"发行" → "小程序-微信"
5. 上传代码
6. 在微信公众平台提交审核
7. 审核通过后发布

**给AI的需求**：
```
生成微信小程序发布检查清单：
1. 代码检查项
2. 配置检查项
3. 审核注意事项
4. 常见被拒原因
```

## 四、小结与思考题

### 本节要点

1. 小程序开发门槛低，适合快速验证想法
2. uni-app一套代码，多端运行
3. 本地存储适合轻量级应用
4. 发布流程简单，审核通过即可上线

### 思考题

1. 小程序和网页应用，各适合什么场景？
2. 如果要添加云同步功能，你会怎么设计？
