## 一、课程导入

### 上节课回顾

你已经学会了：
- 5W2H需求澄清法
- 用户故事编写
- 验收标准定义

**现在，开始写代码。**

---

### 开发策略

不是让AI一次性生成所有代码，而是：
1. 分模块开发
2. 每模块验收后再继续
3. 有问题及时调整

**这是工程化，不是许愿。**

### 2.3 分模块开发

#### 模块1：后端API基础

Task：搭建Express服务器

```markdown
需求：
- 创建Express应用
- 配置中间件（cors, json解析）
- 创建基础路由
- 连接MongoDB

验收标准：
□ 服务器能启动
□ 能响应GET /api/test
□ 能连接数据库
```

#### 模块2：文章API

Task：实现文章CRUD

```markdown
需求：
- POST /api/articles - 创建文章
- GET /api/articles - 获取文章列表
- GET /api/articles/:id - 获取文章详情
- PUT /api/articles/:id - 更新文章
- DELETE /api/articles/:id - 删除文章

验收标准：
□ 所有API能正常工作
□ 数据正确存储到数据库
□ 返回格式统一
```

#### 模块3：前端基础

Task：创建React应用

```markdown
需求：
- 创建React项目
- 配置路由
- 创建基础布局组件

验收标准：
□ 应用能启动
□ 路由能正常工作
□ 布局显示正确
```

#### 模块4：文章列表页

Task：实现文章列表

```markdown
需求：
- 调用API获取文章列表
- 显示文章标题和摘要
- 点击跳转到详情页

验收标准：
□ 能正确显示文章列表
□ 样式美观
□ 点击能跳转
```

#### 模块5：文章详情页

Task：实现文章详情

```markdown
需求：
- 调用API获取文章详情
- 显示完整内容
- 支持Markdown渲染

验收标准：
□ 能正确显示文章内容
□ Markdown渲染正常
□ 样式美观
```

### 2.4 Prompt工程

#### 给AI的Prompt模板

```markdown
# 任务描述
实现[具体功能]

# 上下文
- 项目结构：[描述项目结构]
- 技术栈：[前端/后端技术]
- 已有代码：[相关代码片段]

# 需求详情
[详细描述需求]

# 验收标准
□ [标准1]
□ [标准2]
□ [标准3]

# 约束条件
- [约束1]
- [约束2]
```

---

### 2.5 代码审查

每个模块完成后，问自己：

```
□ 代码能跑吗？
□ 符合需求吗？
□ 有错误处理吗？
□ 有安全问题吗？
□ 性能有问题吗？
```

有问题，让AI改。不要自己上手。

## 四、小结与思考题

### 本节要点

1. 分模块开发，每模块验收后再继续
2. 使用Prompt模板，确保需求传达清晰
3. 代码审查是必须的，不要直接Accept All
4. 有问题让AI改，不要自己上手

### 思考题

1. 分模块开发的好处是什么？你之前是怎么做的？
2. 代码审查时，你最关注哪些方面？
