## 二、核心内容

### 2.1 浏览器插件基础

#### Manifest V3

Chrome扩展的最新标准：

```json
{
  "manifest_version": 3,
  "name": "网页内容收藏",
  "version": "1.0.0",
  "description": "一键收藏网页内容到笔记",
  "permissions": [
    "activeTab",
    "storage"
  ],
  "action": {
    "default_popup": "popup.html",
    "default_icon": {
      "16": "icon16.png",
      "48": "icon48.png",
      "128": "icon128.png"
    }
  },
  "background": {
    "service_worker": "background.js"
  },
  "content_scripts": [
    {
      "matches": ["<all_urls>"],
      "js": ["content.js"]
    }
  ]
}
```

#### 核心概念

| 组件 | 作用 | 场景 |
|-----|------|------|
| popup | 点击图标弹出的窗口 | 快速操作 |
| background | 后台脚本 | 事件监听、数据处理 |
| content script | 注入页面的脚本 | 操作DOM |
| options | 选项页面 | 插件设置 |

### 2.2 项目：网页内容收藏助手

#### 功能概述

一键收藏当前网页：
- 保存标题、URL、选中内容
- 添加标签和备注
- 导出为Markdown
- 同步到Notion（可选）

#### 页面设计

```
Popup页面：
- 当前页面标题（可编辑）
- 当前URL
- 选中的文本内容
- 标签输入
- 备注输入
- 保存按钮
- 查看收藏列表链接

Options页面：
- 导出设置
- Notion集成配置
- 数据管理
```

### 2.3 开发流程

#### 步骤1：项目结构

```
web-clipper/
├── manifest.json      # 插件配置
├── popup.html         # 弹出窗口
├── popup.js           # 弹出窗口逻辑
├── popup.css          # 弹出窗口样式
├── background.js      # 后台脚本
├── content.js         # 内容脚本
├── options.html       # 选项页面
├── options.js         # 选项页面逻辑
├── icons/             # 图标
│   ├── icon16.png
│   ├── icon48.png
│   └── icon128.png
└── lib/               # 工具库
    └── storage.js     # 存储封装
```

**给AI的需求**：
```
创建浏览器插件项目结构：
1. Manifest V3配置
2. popup页面（HTML+JS+CSS）
3. background脚本
4. content脚本
5. options页面
6. 图标占位文件

功能：网页内容收藏助手
```

---

#### 步骤2：Popup开发

**给AI的需求**：
```
开发popup页面：
1. 获取当前标签页的标题和URL
2. 获取用户选中的文本内容
3. 表单包含：
   - 标题输入（默认页面标题，可编辑）
   - URL显示（只读）
   - 选中文本显示（可编辑）
   - 标签输入（支持多个，逗号分隔）
   - 备注文本域
4. 保存按钮：保存到本地存储
5. 成功提示

样式：简洁，宽度400px，适配Chrome popup
```

---

#### 步骤3：数据存储

**给AI的需求**：
```
实现收藏数据存储：
1. 使用chrome.storage.local存储
2. 数据结构：
   {
     id: 时间戳,
     title: 页面标题,
     url: 页面URL,
     content: 选中的内容,
     tags: 标签数组,
     note: 备注,
     createdAt: 创建时间
   }

3. 实现方法：
   - saveClip: 保存收藏
   - getClips: 获取所有收藏（支持按标签筛选）
   - deleteClip: 删除收藏
   - exportClips: 导出为Markdown
```

---

#### 步骤4：内容脚本

**给AI的需求**：
```
开发content脚本：
1. 监听快捷键（Ctrl+Shift+S）
2. 获取选中的文本
3. 高亮显示选中的内容
4. 发送消息给popup

注意：处理跨域问题，处理动态加载的内容
```

---

#### 步骤5：Options页面

**给AI的需求**：
```
开发options页面：
1. 收藏列表展示
   - 搜索功能
   - 按标签筛选
   - 分页显示

2. 导出功能
   - 导出为Markdown文件
   - 导出为JSON

3. Notion集成（可选）
   - 配置Notion Token
   - 选择数据库
   - 同步收藏到Notion

4. 数据管理
   - 备份数据
   - 恢复数据
   - 清空数据
```

### 2.4 发布流程

#### Chrome Web Store发布

1. 打包扩展（zip文件）
2. 注册开发者账号（$5一次性费用）
3. 进入Chrome Web Store开发者后台
4. 上传扩展包
5. 填写商店信息（描述、截图、分类）
6. 提交审核
