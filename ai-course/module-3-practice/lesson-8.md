# 第8课：用AI开发一个网页应用（上）——需求澄清

> 课程模块：实战开发 | 预计时长：60分钟

---

## 一、课程导入

### 一个真实的问题

很多产品想法，死在了"需求不清"。

你告诉AI："帮我做一个记账App。"

AI问："什么平台？功能有哪些？用户是谁？数据怎么存？"

你说："你自己看着办。"

**结果：AI做出了一个你不想要的东西。**

问题不在AI，在需求没澄清。

---

### 需求澄清的价值

需求澄清是开发的第一步，也是最关键的一步。

需求不清，后面全是浪费。

**记住：AI不会读心术。你说不清楚，它就做不对。**

---

## 二、核心内容

### 2.1 需求澄清的5W2H法

| 维度 | 问题 | 示例 |
|-----|------|------|
| What | 做什么？ | 记账App，记录收入和支出 |
| Who | 给谁用？ | 个人用户，年龄25-40岁 |
| Why | 为什么要做？ | 解决记账繁琐的问题 |
| When | 什么时候用？ | 日常生活中，随时记录 |
| Where | 在哪用？ | 手机端，移动场景 |
| How | 怎么做？ | Web App，后端API+前端界面 |
| How Much | 资源和限制？ | 个人项目，预算有限 |

---

### 2.2 需求澄清检查清单

在开始开发前，确保你能回答以下问题：

```
□ 产品定位
  - 这是什么产品？
  - 解决什么问题？
  - 目标用户是谁？

□ 核心功能
  - 必须有的功能（MVP）
  - 可以有的功能（Nice to have）
  - 不需要的功能

□ 技术约束
  - 目标平台？
  - 技术栈偏好？
  - 部署环境？

□ 用户场景
  - 用户如何使用？
  - 典型使用流程？
  - 边界情况？

□ 非功能需求
  - 性能要求？
  - 安全要求？
  - 可扩展性？
```

---

### 2.3 一个完整的案例：记账App

#### 需求澄清过程

**第一轮：粗略需求**

```
我想做一个记账App。
```

**问题**：太模糊，无法开始。

---

**第二轮：补充信息**

```
我想做一个个人记账App，可以记录收入和支出。
```

**问题**：还是不够具体。

---

**第三轮：5W2H澄清**

```
What：个人记账App
- 功能：记录收入、支出、分类统计
- MVP功能：添加记录、查看列表、简单统计

Who：个人用户
- 年龄：25-40岁
- 场景：日常消费记录

Why：解决记账繁琐的问题
- 痛点：现有App太复杂，广告多
- 目标：简单、无广告

Where：Web App
- 平台：浏览器访问
- 设备：手机为主，电脑可用

How：技术方案
- 前端：React
- 后端：Node.js + Express
- 数据库：SQLite（简单起见）

How Much：资源和限制
- 个人项目，预算为0
- 开发周期：2周
```

**结果**：需求清晰，可以开始开发了。

### 2.4 如何让AI理解你的需求

#### 好的需求描述模板

```markdown
# 项目名称：记账App

## 产品定位
一个简单的个人记账工具，解决记账繁琐的问题。

## 目标用户
- 年龄：25-40岁
- 场景：日常消费记录
- 痛点：现有App太复杂、广告多

## 核心功能（MVP）
1. 添加记账记录（金额、类型、分类、备注）
2. 查看记账列表（按日期/分类筛选）
3. 简单统计（月度收支、分类占比）

## 技术方案
- 前端：React + TailwindCSS
- 后端：Node.js + Express
- 数据库：SQLite

## 用户流程
1. 打开App → 看到本月概览
2. 点击"+"按钮 → 添加记录
3. 填写金额、选择类型和分类 → 保存
4. 返回列表 → 查看记录

## 边界情况
- 金额为0或负数
- 分类未选择
- 网络异常
```
