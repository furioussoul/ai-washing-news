## 二、核心内容

### 2.1 需求描述的层次

#### 层次一：一句话需求（Vibe Coding级别）

```
"做个登录功能"
```

**问题**：AI只能猜，结果不可控。

#### 层次二：功能列表（初级）

```
登录功能需要：
- 邮箱输入框
- 密码输入框
- 登录按钮
- 错误提示
```

**问题**：缺少边界情况和验收标准。

#### 层次三：完整需求文档（专业）

```
## 登录功能需求

### 功能描述
用户通过邮箱和密码登录系统

### 输入
- 邮箱：必填，格式验证
- 密码：必填，最少6位

### 流程
1. 用户输入邮箱和密码
2. 点击登录按钮
3. 系统验证邮箱和密码
4. 验证通过，跳转到首页
5. 验证失败，显示错误信息

### 错误处理
- 邮箱格式错误：提示"请输入正确的邮箱格式"
- 密码少于6位：提示"密码不能少于6位"
- 邮箱不存在：提示"邮箱或密码错误"（模糊提示）
- 密码错误：提示"邮箱或密码错误"

### 验收标准
- [ ] 输入正确的邮箱密码，登录成功
- [ ] 输入错误的密码，显示错误提示
- [ ] 邮箱格式错误时，前端校验不通过
- [ ] 登录成功后，30分钟内免重新登录
```

**这才是AI能准确理解的需求。**

### 2.2 需求描述的框架

#### 框架：5W2H + 验收标准

| 维度 | 问题 | 示例 |
|-----|------|------|
| What | 做什么？ | 用户登录功能 |
| Why | 为什么？ | 区分用户，保护数据 |
| Who | 给谁用？ | 网站注册用户 |
| When | 什么时候？ | 访问需要权限的页面时 |
| Where | 在哪里？ | 网站右上角登录入口 |
| How | 怎么做？ | 邮箱+密码，JWT认证 |
| How much | 做到什么程度？ | 支持1000并发，99.9%可用 |

#### 框架：用户故事 + 验收条件

```
作为 [角色]
我想要 [功能]
以便 [价值]

验收条件：
1. [具体可验证的条件]
2. [具体可验证的条件]
3. [具体可验证的条件]
```

### 2.3 常见错误和修正

#### 错误1：过于抽象

**错误**：
```
"做个好看的个人主页"
```

**修正**：
```
个人主页需求：
- 顶部：个人头像和简介
- 中部：技能标签（最多8个）
- 下部：项目展示（3-6个）
- 底部：联系方式和社交媒体链接
- 风格：极简主义，主色调深蓝色
- 响应式：适配手机和桌面
```

#### 错误2：遗漏边界情况

**错误**：
```
"用户可以搜索文章"
```

**修正**：
```
文章搜索功能：
- 支持按标题搜索
- 支持按内容搜索
- 搜索结果按相关性排序
- 空搜索词提示"请输入搜索内容"
- 无结果时显示"未找到相关文章"
- 搜索中显示loading状态
- 支持分页，每页20条
```

#### 错误3：假设AI知道背景

**错误**：
```
"用我们之前讨论的方案"
```

**修正**：
```
使用以下技术方案：
- 前端：React + TypeScript
- 后端：Node.js + Express
- 数据库：PostgreSQL
- 部署：Docker + AWS
```

```

```

#### 技巧2：上下文提供

```

- 这是一个电商网站的前端项目
- 使用Next.js 14 + Tailwind CSS
- 已经配置了shadcn/ui组件库

当前任务：
实现购物车组件...
```

#### 技巧3：示例驱动

```
参考以下示例代码的风格：

```typescript
// 示例代码
export function Button({ children, onClick }: ButtonProps) {
  return (
    <button 
      className="px-4 py-2 bg-blue-500 text-white rounded"
      onClick={onClick}
    >
      {children}
    </button>
  );
}
```

请按照同样的风格实现Input组件...
```

#### 技巧4：分步描述

```
第一步：创建数据模型
第二步：实现API接口
第三步：开发前端页面
第四步：添加表单验证
```

## 四、小结与思考题

### 本节要点

1. 需求描述有层次，从一句话到完整文档
2. 5W2H + 验收标准是需求描述的完整框架
3. 避免抽象、遗漏边界、假设背景
4. Prompt工程：角色、上下文、示例、分步

### 思考题

1. 你之前的需求描述在哪个层次？
2. 如果要描述一个复杂功能，你会用什么框架？
