## 一、课程导入

### 一个效率瓶颈

你同时开了3个AI窗口：
- 窗口1：写前端代码
- 窗口2：写后端API
- 窗口3：写测试用例

**你是整个系统的瓶颈。**

如果能让AI自己协作呢？前端写完自动通知后端，后端写完自动通知测试...

**这就是多Agent协同的价值。**

### 从人工协调到自动协作

传统方式：
```
你 → AI 1 → 你 → AI 2 → 你 → AI 3
```

多Agent协同：
```
你 → AI 1 → AI 2 → AI 3 → 你（验收）
```

**你从"协调者"变成"验收者"。**

## 二、核心内容

### 2.1 为什么需要多Agent协同

#### 单Agent的问题

1. **上下文污染**：一个Agent做所有事，上下文越来越乱
2. **角色混淆**：同一个Agent既写代码又写测试，质量无法保证
3. **效率瓶颈**：串行执行，等待时间长

#### 多Agent的优势

| 维度 | 单Agent | 多Agent |
|-----|---------|---------|
| 上下文 | 容易污染 | 各自独立 |
| 专业性 | 通用 | 专精 |
| 质量 | 难以保证 | 互相制约 |
| 效率 | 串行 | 可并行 |
| 可扩展 | 有限 | 灵活 |

---

### 2.2 Agent角色的设计

#### 基础角色（3个）

```
Developer → Reviewer → Tester
```

- **Developer**：写代码
- **Reviewer**：审查代码
- **Tester**：写测试、执行测试

#### 进阶角色（5个）

```
Architect → Developer → Reviewer → Tester → Deployer
```

- **Architect**：技术设计、架构决策
- **Deployer**：部署、运维

#### 专业角色（7个+）

```
Product Manager → Architect → Frontend → Backend → Reviewer → Tester → Deployer
```

- **Product Manager**：需求分析、产品规划
- **Frontend**：前端开发
- **Backend**：后端开发

---

### 2.3 Agent之间的协作模式

#### 模式一：流水线（Pipeline）

```
Task A → Task B → Task C → Task D
```

适用场景：有明确依赖关系的任务

示例：
```
设计数据库 → 实现API → 编写测试 → 部署上线
```

#### 模式二：并行（Parallel）

```
        ┌→ Task A →┐
Task 0 →┼→ Task B →┼→ Task 4
        └→ Task C →┘
```

适用场景：无依赖关系的子任务

示例：
```
需求分析 →┬→ 前端开发 →┬→ 集成测试
          └→ 后端开发 →┘
```

#### 模式三：评审（Review）

```
Task A → Review → 通过？→ Task B
              ↓ 否
         返回修改
```

适用场景：需要质量把关的环节

### 2.4 自动化工作流设计

#### 触发器类型

| 触发器 | 说明 | 示例 |
|-------|------|------|
| 状态变更 | 任务状态改变时 | Task变为Done时 |
| 时间触发 | 到达指定时间 | 每天上午9点 |
| 条件触发 | 满足特定条件 | 优先级为High时 |
| 手动触发 | 用户手动执行 | 点击按钮 |

#### 动作类型

| 动作 | 说明 | 示例 |
|-----|------|------|
| 创建任务 | 创建新Task | 创建测试任务 |
| 分配任务 | 分配给Agent | 分配给Tester |
| 更新状态 | 修改任务状态 | 标记为In Progress |
| 发送通知 | 通知相关人员 | 发送邮件/消息 |
| 执行脚本 | 运行自动化脚本 | 执行部署脚本 |

#### 示例工作流

```yaml
工作流名称：开发完成自动测试

触发条件：
  类型：状态变更
  任务状态：Done
  任务类型：开发任务

执行动作：
  1. 创建任务：
     标题：测试 {任务标题}
     类型：测试任务
     分配给：Tester Agent
     
  2. 设置依赖：
     阻塞于：原开发任务
     
  3. 发送通知：
     接收者：Tester Agent
     内容：新测试任务已创建，请查收
```
